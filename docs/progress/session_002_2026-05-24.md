## Ce qui a été fait

### Pull des données
- 1000 matchs Master+ EUW pulled via Riot API
- Rate : ~8 matchs/seconde (cache disque après le premier run)
- Durée : ~40 min
- Stockage : `data/raw/` (1001 fichiers JSON : 1001 timelines + 1001 infos match)

### Bug trouvé et corrigé : kills/deaths à zéro
`participantFrames` dans la Riot API timeline ne contient **pas** de champ `kills` ni `deaths` — ces stats viennent uniquement des events `CHAMPION_KILL`. Le parser initial lisait des champs inexistants → diff toujours 0.

**Fix :** on parcourt tous les events `CHAMPION_KILL` jusqu'à `target_min` et on incrémente les compteurs par équipe via `killerId` / `victimId`.

### Dataset
- 4037 snapshots (1001 matchs × 5 timestamps, certains matchs trop courts pour 25/30 min)
- Blue win rate : 47.9% — dataset équilibré ✅
- Distribution timestamps : 10min=1001, 15min=1001, 20min=925, 25min=786, 30min=324

## Résultats — Premier entraînement réel

| Modèle | Accuracy | AUC-ROC | Log-loss | CV AUC |
|--------|----------|---------|----------|--------|
| Logistic Regression | 77.5% | 0.863 | 0.455 | 0.869 |
| XGBoost (calibré) | 77.4% | 0.857 | 0.473 | 0.859 |

**Graphiques :** `models/evaluation.png` (ROC, calibration, feature importances)

### Comparaison 100 vs 1000 matchs

| Dataset | LogReg AUC | XGBoost AUC |
|---------|-----------|-------------|
| 100 matchs | 0.832 | 0.823 |
| 1000 matchs | **0.863** | **0.857** |
| Gain | +3.1% | +3.4% |

### Observation clé
LogReg tient XGBoost de justesse — cohérent avec des features quasi-linéaires (plus l'écart est grand, plus la victoire est probable). XGBoost devrait creuser l'écart à 3000+ matchs en captant les interactions non-linéaires (ex : baron + gold_diff simultanément).
