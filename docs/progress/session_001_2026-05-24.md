## Décisions prises

### Features : 11 au lieu de 9

On a gardé les 9 features du README et ajouté :
- **`gold_diff`** — différence d'économie totale entre les deux équipes. Le gold est le meilleur proxy de l'avantage en LoL (chaque kill, CS, tour vaut un certain gold). Plus prédictif que les kills seuls.
- **`kills_last_3min`** — kills de l'équipe bleue dans les 3 dernières minutes. Capture le **momentum** : une équipe qui vient de faire un teamfight +3 kills est dans une situation très différente d'une équipe avec le même score depuis 10 min.

### Snapshots : 5 timestamps au lieu de 3

10, 15, 20, 25, **30** min — plus de granularité = plus de données d'entraînement par match, et on couvre mieux les parties longues.

### Calibration du modèle XGBoost

XGBoost donne par défaut des probabilités mal calibrées (trop extrêmes). On applique `CalibratedClassifierCV(method="isotonic")` pour que "70% prédit = 70% de winrate réel" dans les données. Crucial pour que la jauge live soit crédible.

## Pipeline créé

```
src/data/pull_matches.py     — Pull Riot API Master+ EUW avec rate limit
src/data/parse_timelines.py  — Timelines → features DataFrame (parquet)
src/features/build_features.py — Source unique de vérité pour les features
src/models/train.py          — LogReg baseline + XGBoost calibré + graphiques
src/models/predict.py        — Inférence + conseils stratégiques
src/live/live_client.py      — Wrapper Live Client API, mapping features train/live
app/main.py                  — Streamlit : gauge, tendance, conseils (polling 7s)
```
